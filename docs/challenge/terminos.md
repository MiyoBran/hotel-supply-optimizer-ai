## Términos de uso

### Finalidad

Este proyecto se entrega con fines educativos y de demostración para el
challenge. El objetivo es mostrar una arquitectura de agente con reglas de
negocio determinísticas y búsqueda semántica.

### Limitaciones

- Los precios y sugerencias dependen de los CSV cargados en el repositorio.
- La lógica de negocio está optimizada para el caso del Hotel Austral.
- El modelo de lenguaje puede cometer errores o interpretar mal una consulta.
- La disponibilidad de Cohere depende del servicio externo y de la API key.

### Responsabilidad de uso

- Verificá siempre los resultados antes de tomar decisiones operativas.
- No uses el agente como única fuente para compras críticas, auditorías o
  compromisos contractuales.
- Si el entorno cambia, actualizá dependencias, modelo y documentación antes de
  volver a usarlo.

### Uso permitido

- Ejecutar el proyecto localmente para aprendizaje o evaluación.
- Adaptar la lógica de negocio a otros catálogos o proveedores.
- Documentar mejoras y resultados de pruebas.

### Uso no recomendado

- Guardar datos sensibles en el prompt.
- Publicar la API key de Cohere en el repositorio.
- Dependender del agente sin revisión humana.

### Aviso

Este proyecto no ofrece garantías de exactitud, disponibilidad ni idoneidad para
uso productivo.
